**发现并修复的 bug（实机测试的价值）：**
| # | 严重度 | 问题 | 根因 | 修复 |

| BUG-1 | 🔴 崩溃 | 切换语言后 App 崩溃（一启动就崩，因语言已持久化） | `ProvideAppLocale` 用 `createConfigurationContext` 替换 `LocalContext`，新 Context 不链回 Activity → `rememberLauncherForActivityResult` 找不到 `ActivityResultRegistryOwner` | 改用 `ContextWrapper(activity)` 只覆盖 `getResources()`，baseContext 仍是 Activity |
| BUG-2 | 🟡 UX | 停止态"未发现可用接口"，无法先选接口再启动 | 接口只在 `start()` 内扫描 | 加 `refreshInterfaces()`（停止态扫描），ViewModel init 调用；勾选状态改从 settings 真源读取 |
| BUG-3 | 🟢 小 | 停止态本地网络权限即使已授予也显示 Problem | 停止态未更新 DiagnosticsSummary 权限字段 | `refreshInterfaces` 同步更新诊断权限项 |

### 真机端到端流量测试（Android 17）— ✅ 全通过
修复后：UI 启动服务 → `ss -ltn` 确认设备上 **`*:8080`/`*:1080`/`*:8899` 均在监听**（FGS + 0.0.0.0 绑定在 API 37 正常）。再经 `adb forward` + 主机 `curl` 走代理拉真实互联网流量（取消勾选接口使准入=放行全部以便 loopback 转发测试）：

| 路径 | 目标 | 结果 |
|---|---|---|
| HTTP 代理（普通转发） | http://example.com | **HTTP 200** ✓ |

| SOCKS5 代理 | http://example.com | **HTTP 200** ✓ |

**结论：HTTP/HTTPS CONNECT/SOCKS5/PAC 四条路径在真机上把真实流量正确中继出去。** 结合 33 个 JVM 测试，代理核心功能已端到端验证。

## 5. 已知限制 / 后续（持续更新）

- **端口/协议开关为"重启生效"**：运行时改端口或开关某协议不会即时重绑 server（仅在下次 Start 生效）。运行时即时生效的是：连接上限/缓冲/并行度、认证开关、出口护栏、mDNS、入口列表。这是 V1 可接受的取舍（动态增删 server 留待后续）。已在 UI 行为上保持一致（设置改动持久化，重启代理后采用）。
- 认证凭据输入 UI + 加密存储（D-E）：仍待补。
- 停止态接口列表为一次性扫描（不随网络变化自动刷新；运行态由 NetworkCallback 维护）。
- mDNS/`ACCESS_LOCAL_NETWORK` 在真机的 NSD 行为、Play 对 FGS 类型审核：见 grounded-reference 的 NEEDS VERIFICATION。
