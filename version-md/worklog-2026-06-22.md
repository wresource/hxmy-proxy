### 模拟器实机自查（Pixel 10 Pro, Android 17 / API 37）— 通过截图 + adb 交互逐屏验证
环境：`Pixel_10_Pro` AVD（**API 37**，顺带验证本地网络权限硬门）。流程：installDebug → 授权 → 启动 → 逐屏 screencap（我直接看图）+ uiautomator 取控件 bounds + `adb input tap` 交互 + logcat 抓崩溃。

**验证通过：**
- Dashboard：状态卡（Stopped/Sharing 切换正确）、底部导航字形、Start/Stop 响应式更新 ✓
- 启动服务：FGS `connectedDevice` 起成功（logcat 确认），接口枚举出 eth0→Ethernet / wlan0→Wi-Fi（分类正确）✓
- Interfaces：勾选 wlan0 → Dashboard 推荐入口实时出现 **`SOCKS5 hxmyproxy.local:1080` + `SOCKS5 10.0.2.16:1080`**（.local + IP 兜底，D1 正确）✓
- Settings：语言/预设/协议/端口/认证布局完整；**中英文运行时切换双向均成功、无崩溃** ✓
- Diagnostics：6 项状态渲染正常 ✓

**发现并修复的 bug（实机测试的价值）：**
| # | 严重度 | 问题 | 根因 | 修复 |
|---|---|---|---|---|
| BUG-1 | 🔴 崩溃 | 切换语言后 App 崩溃（一启动就崩，因语言已持久化） | `ProvideAppLocale` 用 `createConfigurationContext` 替换 `LocalContext`，新 Context 不链回 Activity → `rememberLauncherForActivityResult` 找不到 `ActivityResultRegistryOwner` | 改用 `ContextWrapper(activity)` 只覆盖 `getResources()`，baseContext 仍是 Activity |
| BUG-2 | 🟡 UX | 停止态"未发现可用接口"，无法先选接口再启动 | 接口只在 `start()` 内扫描 | 加 `refreshInterfaces()`（停止态扫描），ViewModel init 调用；勾选状态改从 settings 真源读取 |
| BUG-3 | 🟢 小 | 停止态本地网络权限即使已授予也显示 Problem | 停止态未更新 DiagnosticsSummary 权限字段 | `refreshInterfaces` 同步更新诊断权限项 |

> 已知非 App 问题：截图左侧 Gmail 浮窗是模拟器的系统气泡通知，会遮挡部分 UI，与本 App 无关。
