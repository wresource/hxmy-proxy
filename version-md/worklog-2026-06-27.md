## 排查(实测结论,无功能代码改动)

- **「release 翻墙服务全 timeout」**:3 个 agent 反编译 release DEX + 真机实测 → **证伪**。
  `RuleAction` 未被 enum-unbox、判定链/`bypassVpn` 守卫完好,claude.ai 判 `PROXY`、经 VPN `200`、24 并发全 OK。
  图标矢量逐字节匹配源码 → release 是最新代码、非缓存。**根因:更早构建的旧包(缺 race fix `e573824`),非 R8 优化 bug**。
  诊断期临时加的 `-dontoptimize` 已回退。
- **访问慢**:`ping` 证 VPN 出口纯 RTT = 200ms(8.8.8.8 / claude.ai 同);经代理 ≈ 手机直连(转发开销 <10ms);
  10 并发 0.84s(relay 池零排队、并行有效);连接复用 0.22s vs 全新握手 0.94s;HTTP/3 代理转不了。
  **瓶颈 = VPN 出口 RTT,代理与并行度都清白**,唯一杠杆是换低延迟节点。
